# 4桁数字パズル Webアプリ 仕様書

## 1. 概要

### 1.1 プロダクト概要
4桁の数字に対し、桁の入れ替え・四則演算・累乗を用いて、**21以下でなるべく21に近い値**を作るパズルゲームの解探索Webアプリ。

### 1.2 ゴール
- ユーザーが入力した4桁数字に対し、考えうる全ての演算パターンを網羅的に列挙
- 21に最も近い解を即座に提示
- 全ての結果値とその生成式をブラウズ可能にする

### 1.3 対象ユーザー
- 数字パズル愛好者
- ゲーム中の確認・自己採点用途のユーザー

---

## 2. 機能要件

### 2.1 入力
| 項目 | 仕様 |
|---|---|
| 形式 | 4桁の数字文字列(`0000` 〜 `9999`) |
| 入力UI | 数字専用テキストフィールド(`inputmode="numeric"`) |
| バリデーション | 4桁未満は左0埋め、5桁以上は不可、数字以外は不可 |
| 即時反映 | 入力確定(Enter or ボタン)で計算開始 |

### 2.2 計算
- 桁グループ分けパターン(全5種、`(4)` は演算なしのため除外)
  - `(1,1,1,1)`: 1桁 × 4
  - `(2,1,1)`: 2桁 × 1 + 1桁 × 2
  - `(2,2)`: 2桁 × 2
  - `(3,1)`: 3桁 × 1 + 1桁 × 1
- 演算子: `+`, `-`, `*`, `/`, `^`
- 括弧の組み合わせは Catalan 数に従い全列挙
- 重複式は除外
- 最大演算パターン数: **18,960**(全桁異なる場合)

### 2.3 結果表示
| 表示項目 | 内容 |
|---|---|
| ベスト解 | 21以下で最大の値とその生成式(複数あれば全て) |
| 21ちょうど | 21を生成する式があれば最優先で表示 |
| 全結果一覧 | 結果値の降順 / 昇順切替、値ごとにグループ化 |
| 21への近接度 | 各値に「21との差」を併記 |
| 式の表示 | 等幅フォント、括弧と演算子は色分け |

### 2.4 操作機能
- 結果値による検索(例: `21` を入力でその結果に絞り込み)
- 演算子フィルタ(例: 累乗を含む式のみ表示)
- 式のクリックでクリップボードにコピー
- 共有用URL生成(クエリパラメータに input を埋め込む)

### 2.5 ゲームモード(任意拡張)
- ランダム4桁生成
- ユーザーの解答式を入力 → 評価
- 21との差で採点
- 「正解(21)」が存在する問題かをタイマー後に開示

```typescript
// key:   計算結果の整数値
// value: その値を生成する式文字列の配列(重複除去済み)
```

```typescript
interface AppState {

    sortOrder: 'asc' | 'desc';
  };
```

## 5. 計算ロジック仕様

### 5.1 演算子定義
| 演算子 | 動作 | 除外条件 |
|---|---|---|
| `+` | 加算 | なし |
| `-` | 減算 | なし |
| `*` | 乗算 | なし |
| `/` | 除算 | 除数が 0 |
| `^` | 累乗 | `0^(≤0)`、負数の非整数乗、指数絶対値 > 30 |

### 5.2 結果フィルタ
- 結果が `NaN` または `Infinity` → 除外
- 結果が非整数 → 除外
- 結果絶対値 > 10⁹ → 除外

### 5.3 アルゴリズム
1. 入力文字列を桁配列に変換(例: `"2201"` → `[2, 2, 0, 1]`)
2. 桁配列の重複除去順列を生成
3. 各順列に対し、5種のグループ分け × 各並び × 各括弧 × 各演算子の組合せで式を生成
4. 各式を評価し、結果が有効ならば `Map<number, Set<string>>` に追加
5. 完成後、結果値の降順にソート

## 9. テスト要件

### 9.1 単体テスト(`solve.ts`)
- 既知の入力に対する期待結果の検証
  - `2201` → 21 を含むこと、`22-01` 相当の式が存在すること
  - `3201` → 21 を含むこと、`31-10` 相当の式が存在すること
- エッジケース
  - `0000` → 0 のみ
  - `9999` → 期待される結果集合
- ゼロ除算・累乗エッジが除外されることの確認

### 9.2 結合テスト
- 入力 → 計算 → 結果表示の一連フロー
- 不正入力時のエラー表示

## 11. 制約事項・既知の課題

- 単項マイナス(例: `-2 + 23 = 21`)は仕様外
- 階乗・対数等の追加演算は仕様外
- 結果が非整数の解(例: `1/3` を経由して整数になる場合)は中間値が非整数でも最終整数なら採用される
- 同じ値を生成する式が大量にある場合の表示パフォーマンス(ページネーションで対応)
